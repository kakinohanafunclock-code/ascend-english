## 2026-06-23 — 機能拡張（学習時間・単語学習・添削詳細化）
- **決定**: 総学習時間は「各完了タスクの想定所要時間 (estimatedMinutes) の合算」とする（実測経過時間は使わない）。
- **理由**: ユーザー要望。起動時間に依存せず、完了した学習量を安定的に反映するため。値は Reading/Listening=15分、Speaking=10分、Writing=20分。
- **決定**: 英単語学習は Leitner 方式の SRS（box 1–5、間隔 1/2/4/8/16 日）。`good` で昇格・先送り、`again` で box1 へ降格し当日再出題。
- **理由**: 実装が単純で説明可能、外部依存ゼロ。`src/domain/vocab.ts` に純関数として実装しテスト。
- **決定**: AI 添削レスポンスに `translationJa`（日本語訳）と `glossary`（語注: 単語＋日本語意味＋例文）を追加。採点画面で日本語訳を表示し、語注からワンタップで単語帳に追加可能。
- **理由**: ユーザー要望（解説の丁寧化・日本語訳表示・わからない単語を単語帳へ）。フォールバック時は glossary 空配列で安全に動作。
- **決定**: AI への指示（system/prompt）で strengths/improvements/解説を日本語で返すよう明示。
- **理由**: 日本人学習者向けに解説の可読性を高めるため。

## 2026-06-23 — UI / 通知 / デプロイ
- **決定**: 外部フォント CDN (rsms.me Inter) への依存を削除し、システムフォントスタックにフォールバック。
- **理由**: オフライン/サンドボックス環境でのページ読み込みハングを回避し、外部依存ゼロに近づけるため。
- **決定**: デザイントークンの色は `var()` 直値で定義し、Tailwind の不透明度修飾子 (`/30` 等) は使わない。
- **理由**: `<alpha-value>` 形式でないと不透明度修飾子はビルドエラーになるため。必要な淡色は専用トークン (accent-soft 等) で表現。
- **決定**: AI プロキシは Vercel (`api/ai.ts`) と Netlify (`netlify/functions/ai.mts`) の両対応。フロントは `/api/ai` に POST。
- **理由**: 無料枠の両ホスティングでキーをサーバ側に秘匿しつつ動かすため。
- **決定**: 端末を閉じている間の通知は VAPID 設定時のみ Web Push。未設定時はアプリ/SW 稼働中のローカルスケジュール。
- **理由**: コスト0を優先。Push サーバ運用は任意拡張とする。

## 2026-06-23 — 初期技術選定とプロジェクト方針
- **決定**: Vite + React + TS(strict) + Tailwind + Vitest + Testing Library + Playwright を採用。
- **理由**: 要件で明示。Vite/Vitest は同一設定で dev/test を回せ TDD と相性が良い。
- **決定**: アクセント色は紫/青を避け、ink-teal (`#0f6e63`) 1色に限定。配色・余白・タイポは CSS 変数トークン化。
- **理由**: 「生成AIアプリっぽい」派手な見た目を禁止する要件。Linear/Stripe 系の抑えた品位を狙う。
- **決定**: 永続化はまず IndexedDB（localStorage フォールバック）。Supabase は将来オプション。
- **理由**: コスト0・オフライン動作。クラウド同期は必要時に追加。
- **決定**: 音声は Web Speech API (TTS/STT) のみ。音声ファイルは保持しない。
- **理由**: コスト0の要件。Listening は本文→TTS、Speaking は STT で実現。
- **決定**: AI は Claude の安価モデル（`claude-haiku-4-5` 既定）。本番は proxy 関数経由でキー秘匿、応答はキャッシュ。
- **理由**: コスト最小化とキー漏洩防止。ブラウザ直叩き(direct)はローカル検証限定。
- **決定**: スコアは TOEFL 各技能 0–30、合計 0–120 のレンジで推定し、診断は IRT 風の単純な加重で算出。
- **理由**: 公式アルゴリズムは非公開のため、説明可能で安定した近似を採用（README に明記）。
